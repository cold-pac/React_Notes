ReactJs = an open source JavaScript library, for creating user interfaces. Created and maintained by Facebook.
Uses its own markup language 'JSX' - lets your write readable HTML directly within JavaScript.
To write vanilla JavaScript within JSX use curly brackets: 
                
                    { //blah blah blah this is some JavaScript }
JSX code must be compiled into JavaScript. Use a 'transpiler' like Babel.
To create React element: 
                
                    const bro = <div> Hello World! </div>

                    const bro = (
                        <div>
                            <h1>This is a block of JSX</h1>
                            <p>Here's a subtitle</p>
                        </div>
                    );
                    //best practice to use parentheses

To add comments use {/* */}
You need to render your JSX to the HTML DOM using React's rendering API - ReactDOM. ==> contains a .render() method you can use to render React elements to the DOM like this ReactDOM.render(componentToRender, targetNode). componentToRender = React element you want to render, targetNode = DOM node that you want to render the component to. Must be declared after the JSX element declarations (like how you must declare variables before you use them)! 
                
                    const bro = (
                        <div>
                            <h1>Hello World</h1>
                            <p>Lets render this to the DOM</p>
                        </div>
                    );

                    ReactDOM.render(bro, document.getElementById('target'));

You cannot use "class" to define HTML classes in JSX (class is a reserved for object constructor functions in JavaScript). Use className instead (remember camelCase for everything) 
                
                    const bro = (
                        <div className = 'myDiv'>
                            <h1>Add a class to this div</h1>
                        </div>
                    );

In JSX, every tag can be self closing i.e. instead of <div> </div> you can just use <div />
Everything in React is a 'component'. React 'components' are like JS functions -> accept inputs (called "props") and return React elements.
There are two ways to create a React component.
1) use a JavaScript function ==> creates a 'functional' component. 'Stateless' because it can receive data and render it, but does not manage or track changes to that data. Remember: functions in React must begin with a capital! 
                
                    const DemoComponent = function() {
                        return (
                            <div className='customClass' />
                        );
                    };

2) Using the ES6 class syntax (for object constructor functions): 
                
                    class Kitten extends React.Component {
                          constructor(props) {
                            super(props);
                          }

                          render() {
                            return (
                              <h1>Hi</h1>
                            );
                          }
                        }
Class constructor calls super(), which calls the constructor of the parent class (React.Component). Constructor = special method called during the initialisation of objects created using the class syntax. Best practice to call a component's constructor with super, and pass props to both. Makes sure the component is initialized properly.
COMPONENT COMPOSITION:
When you have created many components (created in either way - as simple, stateless functional components or as ES6 class components) - you can create parent components that renders other components as children - by including the child component's name as a tag in the render method of the parent. E.g. 
        
            const ChildComponent = () => {
              return (
                <div>
                  <p>I am the child</p>
                </div>
              );
            };

            class ParentComponent extends React.Component {
              constructor(props) {
                super(props);
              }
              render() {
                return (
                  <div>
                    <h1>I am the parent</h1>
                    <ChildComponent/>
                  </div>
                );
              }
            };
Think about your user interface in terms of components - break your UI down into building blocks
To actually use components (i.e. render React components to the DOM by making a call to the ReactDOM API) - set a React element as = to the Component, like let my-element = < component-name /> ReactDOM.render(element, document.getElementById("target")); or just go ReactDOM.render(< component-name />, document.getElementById("target"))
PROPS: you can pass "props" (properties) to child components. They can have any name you like, be any type of variable (a string, an array, an object) e.g. if you have a parent App component that renders a child component Welcome, you can pass the child component a user property 
        
            const Welcome = (props) => <h1>Hello, {props.user}!</h1>
            //standard to call this props

            //render () {....
            <App>
                <Welcome user = 'Mark' />
            </App>


            const ChildComponent = (props) => <p>{props.colors.join(', ')}</p> {//array methods can be called on the property when you access it}

            <ParentComponent>
              <ChildComponent colors={["green", "blue", "red"]} /> {//to pass an array as a prop, it must be treated as JavaScript and wrapped in curly brackets}
            </ParentComponent>
An example: 
        
            const List= (props) => {
              return <p>{props.tasks.join(", ")}</p>
            };

            class ToDo extends React.Component {
              constructor(props) {
                super(props);
              }
              render() {
                return (
                  <div>
                    <h1>To Do Lists</h1>
                    <h2>Today</h2>
                    <List tasks = {["join the gym", "join the army", "go to dinner"]}/>
                    <h2>Tomorrow</h2>
                    <List tasks = {["walk dog", "wonder about life", "do a pullup"]}/>
                  </div>
                );
              }
            };
To use default props: only assigned "if necessary" i.e. you haven't manually set it in the parent component - by accessing the component's defaultProps property (which is itself an object) e.g. 
        
            const ShoppingCart = (props) => {
              return (
                <div>
                  <h1>Shopping Cart Component</h1>
                </div>
              )
            };
            ShoppingCart.defaultProps = {items: 0};
